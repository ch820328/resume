Firmware Configuration & NVRAM Persistence Validation

Systems Automation & Reliability
Challenge: Validating BIOS settings (Boot Options, Setup configurations) across firmware updates. Traditional UI-based testing was too slow and impossible to apply for In-Band flash validation, where the presentation layer is unavailable.
Solution: Developed a UEFI Shell suite (EDK II/C) to directly manipulate NVRAM variables. Integrated an OOB (Out-of-Band) recovery flow via BMC Redfish to automatically re-flash and recover systems in case of flash failures, ensuring continuous test cycles.
Impact: Automated 80% of regression cases. Enabled In-Band BIOS validation that was previously un-testable via UI. Ensured configuration persistence across hundreds of firmware update cycles, significantly improving release reliability.
